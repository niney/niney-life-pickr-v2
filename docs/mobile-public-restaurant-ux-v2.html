/restaurants-v2 — 네이버 지도식 3-snap 바텀시트로 모바일 UX 재구성. 데스크톱(xl+) 은 기존 3-column 그대로.

🎯01 TL;DR

★ 핵심 5가지
  • 3-snap 시트 (peek / half / full) — @use-gesture/react 드래그, 220ms transform animation
  • Dual-mode 시트 (fixed ↔ scroll) — full 정착 시 position:static swap → body 스크롤 활성 → 모바일 주소창 자동 minify
  • 단일 sticky 통합 헤더 — PublicTopBar 의 root <header> 안에 검색·카테고리 행을 subBar slot 으로 주입
  • ResizeObserver 헤더 실측 — 시트 topOffset 을 상수가 아닌 실측 px → subBar 변동/숨김에 자동 동기화
  • viewKey per-view 스크롤 메모리 — list↔detail 전환 시 list scroll 자동 저장/복원

🧭05 핵심 결정

5-1. 모바일 주소창 minify 메커니즘 (제약 사실)

⚠ 브라우저 제약
주소창은 body(window) 스크롤이 아래로 움직일 때만 줄어든다. JS 강제 불가 (window.scrollTo(0,1) 트릭은 iOS 8+ 부터 무효). transform 드래그 / 내부 div overflow 스크롤은 효과 X.

⇒ 주소창을 줄이려면 그 순간엔 body 가 실제로 스크롤되는 구조여야 한다. 이게 dual-mode 의 존재 이유.

5-3. 한 몸 sticky 통합 (v2 의 결정적 fix)

✕ 초기 버그
snap=full collapse 시 브라우저가 주소창 재표시 → dvh jump → 분리되어 있던 두 sticky(TopBar / SearchRow) 의 paint 가 한 프레임 어긋나 겹침 또는 TopBar 사라짐.
✓ 해결
PublicTopBar 에 subBar slot prop 추가 → root <header> 안에 두 번째 row 로 렌더. 단일 sticky element 가 되어 dvh 변동이 한 단위로 일어남.

headerHeight 는 PublicTopBar 의 headerRef 에 ResizeObserver 부착해 실측. subBar 가 등장/사라지거나 컨텐츠가 바뀌면 즉시 동기화.

🖥08 데스크톱 (xl+) 분기

같은 페이지 컴포넌트 안에서 CSS 분기:

  • <div className="hidden xl:flex"> — 데스크톱 3-column (기존 RestaurantsPage 동일): 좌 PublicRestaurantList + 중 <Outlet/> + 우 PublicRestaurantsMap

  • subBar 컨테이너에 xl:hidden → 데스크톱은 display:none → headerHeight 자동으로 56 (TopBar 만)
  • BottomSheet 는 xl- 영역 안에서만 mount → 데스크톱은 시트 무관

🎛10 BottomSheet 동작 세부

props

interface Props {
  children: React.ReactNode;
  snap?: Snap;                 // 'peek' | 'half' | 'full' — 외부 제어
  onSnapChange?(next: Snap): void;
  peekHeight?: number;         // peek 일 때 최소 시각 높이 (px)
  halfRatio?: number;          // half 비율 (vh - topOffset 대비)
  topOffset?: number;          // 시트가 침범 못하는 상단 영역 (px)
  viewKey?: string;            // 컨텐츠 식별자 — per-view scroll 메모리
}

두 useLayoutEffect

① mode + viewKey 변화 effect
  1. viewKey 가 바뀌었으면: outgoing scroll 저장, 새 키 scroll 로드
  2. mode === 'fixed': html overflow 잠금, inner.scrollTop = scrollPosRef
  3. mode === 'scroll': html overflow 해제, window.scrollTo(0, scrollPosRef)
② snap → mode reconciliation effect
  • 드래그 중이면 skip
  • snap=full + mode=fixed → setTimeout 220ms 후 setMode('scroll')
  • snap≠full + mode=scroll → 즉시 setMode('fixed')

드래그 처리 (useDrag)

  • 핸들에만 바인딩 ({...bind()} on handle div)
  • filterTaps: true, axis: 'y'
  • first 이고 mode === 'scroll' 이면 beginDragFromScroll(): scrollY 저장 후 setMode('fixed')
  • scrollY 가드 없음 — 핸들은 sticky 라 항상 받음
  • last 이면 pickSnap 으로 가장 가까운 스냅 (velocity 가속)

✅14 검증 시나리오 (실기기)

  1. peek → drag up → half → drag up → full: 시트 부드럽게 확장, full 정착 후 220ms 내 mode swap
  2. full + body 스크롤: 주소창 자동 minify
  3. full + 스크롤된 상태 + 핸들 collapse: scroll→fixed swap, body scrollY 0 reset, inner.scrollTop 에 이전 값 미러
  4. 카드 클릭: snap=half, viewMode='detail', subBar 사라짐, detail content top 부터
  5. detail body 스크롤 후 닫기: list 복귀 시 진입 전 snap 복원 + scroll 위치 복원
  6. 주소창 minify 변동: visualViewport resize 가 vh state 갱신 → 시트 위치 즉시 따라감
  7. 데스크톱 폭 (xl+): 시트/subBar 안 보임, 기존 3-column 동작

⚖15 알려진 트레이드오프

  • fixed 모드 inner 직접 스크롤은 메모리에 안 반영 — peek/half 에서 inner 컨테이너를 직접 스크롤한 경우, 다음 mode 전환 시 scrollPosRef 가 덮어씀
  • subBar 등장/사라짐 transition 없음 — detail 진입 시 헤더 영역 즉시 변동. 어색하면 height transition 추가
  • scrollPosRef 가 'list'/'detail' 두 키만 — 식당별 분리 메모리 안 함
  • PublicTopBar root 가 flex → block — 두 번째 row 가 세로 stack 가능하도록. xl+ 에서 시각적 영향 없음
  • React 19.1.0 다운그레이드 — workspace 일관성 위해. 19.2 신규 API 사용처 현재 없음
