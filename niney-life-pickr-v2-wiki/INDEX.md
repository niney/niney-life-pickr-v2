# niney-life-pickr-v2 Knowledge Base

Last compiled: 2026-05-25
Total topics: 16 | Total concepts: 8 | Mode: codebase

선택을 대신 골라주는 서비스 — pnpm + Turborepo 모노레포(Fastify API + Vite 웹 + Expo 앱)의 컴파일된 위키. 처음 본다면 [project-overview](topics/project-overview.md) → 관심 토픽 순서로 읽는 것을 권장. 사용자 대상 공개 페이지(`/`, `/restaurants`)와 어드민 운영 도구(`/admin/restaurants`, `/admin/discover`, `/admin/auto-discover` 등) + 정산하기(`/restaurants/:placeId/settle/new`, `/me/settlements`, `/me/contacts`, `/share/settlements/:token`) 가 한 SPA 안에 분리되어 있다.

> **용어**: 이 위키는 **웹**(`apps/web`) / **앱**(`apps/mobile`) / **모바일**(=웹의 작은 화면)을 구분해서 쓴다. 슬러그·디렉터리·스크립트 식별자는 `mobile`/`web` 그대로. 자세한 규칙은 [schema.md Terminology](schema.md#terminology--웹--앱--모바일).

## How to navigate

- **새 작업 시작** → [project-overview](topics/project-overview.md)에서 디렉터리 지도와 워크플로 파악
- **정산하기** (`/restaurants/:placeId/settle/new` 4단계 stepper / `/me/settlements` 이력 / `/me/contacts` 단골 관리 / `/share/settlements/:token` 비인증 공유 보기) → [settlement](topics/settlement.md) (도메인 전체 — extraction · session · contact · share · edit)
- **백엔드 작업** → [friendly](topics/friendly.md), 크롤링 관련이면 [crawl](topics/crawl.md), LLM 관련이면 [ai](topics/ai.md), 지도 키 관리면 [map](topics/map.md), 정산이면 [settlement](topics/settlement.md)
- **어드민 자동 발견** (`/admin/auto-discover` — 키워드 한 줄 + 카테고리 칩 + targetCount → AI 가 검색어 8개 생성 → 다중 검색 + dedupe → 그룹 5개씩 직렬 크롤·등록) → [auto-discover](topics/auto-discover.md) (전체 흐름·정책·실패 모드) → [crawl](topics/crawl.md) (`startCrawl` 재사용) → [shared](topics/shared.md) (`useAutoDiscoverJob` 훅) → [web](topics/web.md) (페이지/폼/카드)
- **어드민 발견 페이지** (`/admin/discover` — 네이버 PC 지도 검색 + 다중 선택 일괄 크롤링 + 좌/우 패널 토글) → [web](topics/web.md) (페이지·컴포넌트·panelPrefsStore) → [crawl](topics/crawl.md) (HTTP 검색 어댑터 + searchPlaces 서비스 + GET /crawl/search) → [api-contract](topics/api-contract.md) (NaverSearchResult/CrawlSearchQuery/Result) → [shared](topics/shared.md) (`useNaverSearch`)
- **공개 사용자 페이지** (`/`, `/restaurants` — 풀 뷰포트 + 5탭 상세 + 지도) → [web](topics/web.md) → [shared](topics/shared.md) (공개 훅 `useRestaurantsPublic` 등) → [friendly](topics/friendly.md) (공개 라우트 3개) → [map](topics/map.md) (vworld 통합)
- **모바일 UX 작업** (sticky 트랩, body 스크롤, IME, 라이트박스 스냅, iOS 줌 등) → `docs/mobile-public-restaurant-ux.md` (SSOT) → [web](topics/web.md) Key Decisions "모바일 UX 규율" 섹션 → 8개 규칙 적용 사례 확인
- **메뉴 분석 도메인** (분석 → 식당별 그룹핑 → 전역 머지 → 카테고리) → [menu-grouping](topics/menu-grouping.md) → [analytics](topics/analytics.md). 운영은 admin "AI 분석 관리" 페이지([web](topics/web.md))에서.
- **맛집 도메인** (DB 영속화 + 다중 크롤 + AI 요약 + 미디어 프록시) → [crawl](topics/crawl.md) → [friendly](topics/friendly.md) (restaurant/summary/media 모듈) → [shared](topics/shared.md) → [web](topics/web.md) 순으로 흐름 따라가기
- **다이닝코드/캐치테이블 등록** (정식 검색 + 등록 배지 + SSE 일괄 저장) → [web](topics/web.md) `AdminDiningcodePage` → [shared](topics/shared.md) `useDiningcodeBulkSaveJob`/`useDiningcodeRegistered` → [crawl](topics/crawl.md) `diningcode-bulk-save-registry.ts` + adapters → [api-contract](topics/api-contract.md) `DiningcodeBulkSaveJob*` + `CatchtableShopData`
- **출처 가로지르기 같은 가게 묶기** (Naver/다이닝코드/캐치테이블 한 canonical) → [canonical](topics/canonical.md) (후보 매칭·merge/split·자동 검토 큐·**C안 자동 머지 후크**·canonical 단위 삭제·`restaurant.merge.ts` 융합 헬퍼) → 임계 못 넘으면 검토 큐 fallback → 어드민 [web](topics/web.md) `MergeProposalQueue`/`CanonicalMergePanel`
- **vworld 지도 작업** (어드민 단일 마커 / 공개 다중 마커 / WMTS 키 관리 / 카테고리별 라인 아이콘 8종) → [map](topics/map.md) — OpenLayers + WMTS 직접, JS SDK 거부 이유, declutter 해제 정책
- **웹/앱 변경** → [web](topics/web.md) 또는 [mobile](topics/mobile.md), 로직 공유 시 [shared](topics/shared.md)
- **앱 운영 빌드** (Release/EAS/실기기 절차) → [mobile](topics/mobile.md) `apps/mobile/docs/production-build.md` + `.env.production` 자동 로드
- **앱 Metro / Expo Web / React 버전 충돌** → [mobile](topics/mobile.md) Architecture › Metro 모노레포 설정 + Expo Web 섹션
- **스키마 추가** → [api-contract](topics/api-contract.md) (반드시 여기에 zod로 — CLAUDE.md 규칙). 권한 페어로 분리해야 하면 [public-admin-route-split](concepts/public-admin-route-split.md) 패턴 (두 결: 어드민/공개 + 소유자/토큰 보유자)
- **LLM 프롬프트/스키마 변경** → [versioned-llm-prompts](concepts/versioned-llm-prompts.md) — `*_VERSION` 상수 올리고 stale 데이터 처리 흐름 확인. 6 인스턴스 (summary v4 / menu-grouping / global-merge v2 / auto-discover / EXTRACTION — 뒤 두 개는 얇은 인스턴스)
- **`@repo/*` import 에러로 막힐 때** → [workspace-package-resolution](concepts/workspace-package-resolution.md)의 디버깅 순서 따라가기
- **SSE/캐시 머지 패턴** → [stream-driven-cache-merge](concepts/stream-driven-cache-merge.md), 동시성 제어는 [in-memory-singleton-gates](concepts/in-memory-singleton-gates.md)
- **횡단 패턴** → [concepts/](concepts/) 디렉터리 — Zod SSOT 인프라, 권한 라우트 페어 분리, SSE 인증, UI 플랫폼 분기, workspace 패키지 해결, 스트림 캐시 머지, 인메모리 게이트, LLM 프롬프트 버전
